## Техническое задание для стажёрского проекта

Написать RESTful Web API для работы с профилем пользователя.
Должны быть реализованы следующие сервисы:
- Регистрация
 - POST /profile
- Получение профиля по уникальному идентификатору
 - GET /profile/{id}
 - При получении профиля также должен возвращаться полный возраст пользователя
- Обновление профиля
 - PATCH /profile/{id}
 - PUT /profile/{id}
- Удаление профиля
 - DELETE /profile/{id}

Профиль пользователя состоит из следующих полей:
- Логин - строка
- Пароль - строка
- Фамилия - строка
- Имя - строка
- Отчество - строка
- Дата рождения - datetime
- Опыт работы - число

В качестве веб фреймворка использовать fastapi, приложение запускается через asgi uvicorn 
(пока что запускать с одним воркером). Данные хранить в виде словаря в глобальной области видимости.
Все запросы и ответы должны быть структурированы в виде pydantic моделей. Данные с запросов 
должны валидироваться.
При попытке совершить действие с несуществующим профилем должна возвращаться 404 ошибка.
На данном этапе сервисы нужно писать синхронными, однако в дальнейшем они будут доработаны
и сделаны асинхронными. 

Проект в дальнейшем будет дорабатываться, так что сразу подумайте о грамотной структуре 
приложения. Подумайте как лучше расположить различные модули и пакеты, чтобы масштабирование 
приложения происходило без серьёзных проблем.

Дополнительно можно настроить сохранение словаря в json формате на жёсткий диск для
 сохранения данных между перезапусками приложения. Соответственно при запуске приложения 
 этот json файл должен считываться.
